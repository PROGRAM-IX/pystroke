Class Hierarchy

  • pystroke.behaviour.Behaviour: Stores a modular behaviour that can be added to a game entity
  • pystroke.behaviour_engine.BehaviourEngine: Processes all behaviours in beh_dict when update() is called
  • pystroke.draw_engine.DrawEngine: Abstracts the calls to Vex.draw() and other drawing methods
  • pystroke.event_engine.EventEngine: Reads the event queue and passes events to other engines
  • pystroke.game.Game: Container and manager for GameEngine instances
  • pystroke.game_engine.GameEngine: Generic class to contain all logic for the basic running of the game
  • pystroke.hud.HUD: A heads-up display, which comprises various visual elements displayed on a screen to give information to a player
  • pystroke.hud.HUDElement: Generic part of a heads-up display
    • pystroke.hud.HUDLine: An element of a heads-up display consisting of a line
    • pystroke.hud.HUDPolygon: An element of a heads-up display consisting of a polygon
    • pystroke.hud.HUDText: An element of a heads-up display consisting of text
  • pystroke.input_engine.InputEngine: Receives input events from an EventEngine and uses them to maintain an up- to-date keyboard/mouse state
  • pystroke.vector2.Vector2: A two-dimensional vector
  • pystroke.vex.Vex: Vector sprite class (consider renaming) - consists of a list of points which are rendered relative to an x and y at draw time
